# NavLink 授权与激活系统说明 (NavManage)

NavLink 2.0 采用集中式的在线授权验证系统 (NavManage)，不再使用旧版的本地离线签名。所有授权验证、插件市场下载均通过 NavManage 进行。

## 一、系统架构

```
┌─────────────┐       API (HTTPS)       ┌──────────────────┐
│   NavLink   │ <─────────────────────> │    NavManage     │
│  (客户端)    │                         │  (授权/验证服务)  │
└─────────────┘                         └──────────────────┘
   ↑                                             ↑
   │ 1. 输入激活码 + 邮箱                           │ 2. 验证并绑定指纹
   │ 3. 定期在线验证授权状态                         │ 4. 返回授权 Token
```

## 二、部署与配置

### 1. 客户端配置 (NavLink)

系统默认已配置连接到官方/默认授权服务器 (`https://auth.webxx.top`)。通常情况下，您**无需**进行额外配置。

**自定义私有部署：**
如果您部署了自己的 NavManage 服务，或者需要修改连接地址，可以通过 `.env` 覆盖默认配置：

```env
# [可选] 覆盖默认授权服务器地址
AUTH_SERVER_URL=https://您的域名

# [可选] 覆盖默认插件市场地址
PLUGIN_REGISTRY_URL=https://您的域名/api/registry.json
```

### 2. 服务端管理 (NavManage)

NavManage 提供了完整的后台管理界面，用于生成激活码和管理用户。

#### A. 生成激活码
管理员登录 NavManage 后台：
1. **创建用户**: 在用户管理页面创建新用户，系统会自动生成一个激活码。
2. **批量生成**: 可批量生成一组“未绑定”的激活码，用户激活时自动绑定其邮箱。

#### B. 激活码格式
激活码格式为 `XXXX-XXXX-XXXX` (由字母和数字组成)。

## 三、激活流程

### 步骤 1：获取激活码
管理员通过 NavManage 生成激活码，并将其发送给用户。

### 步骤 2：用户激活
1. 登录 NavLink 后台管理界面。
2. 进入 `系统设置` -> `授权信息`。
3. 点击 **"激活授权"**。
4. 输入 **邮箱** 和 **激活码**。
5. 点击提交。

### 步骤 3：系统验证
- NavLink 将向 `AUTH_SERVER_URL` 发送激活请求。
- NavManage 验证激活码有效性，并将其与当前服务器的 **机器指纹 (Fingerprint)** 绑定。
- 激活成功后，NavLink 会保存授权 Token (Auth Token) 到本地数据库。

## 四、常见问题与维护

### Q: 激活失败 "404 Not Found"
- **原因**: 授权服务器地址配置错误或 API 路径不匹配。
- **解决**: 检查 `.env` 中的 `AUTH_SERVER_URL` 是否正确，且没有任何拼写错误（应指向 NavManage 根域名，如 `https://auth.webxx.top`）。

### Q: 激活失败 "Connection refused"
- **原因**: NavLink 容器无法连接到 NavManage 服务器。
- **解决**: 检查网络连通性。如果在同一台机器上通过 Docker 部署，可能需要配置 `extra_hosts` 或使用宿主机 IP。

### Q: 提示 "激活码已被使用"
- **原因**: 该激活码已在另一台机器上激活。
- **解决**: 
    - 如果是用户迁移服务器，请在 NavLink 后台申请 **"迁移码"** (Request Migrate)。
    - 或者管理员在 NavManage 后台生成新的激活码。

### Q: 提示 "您已达到最大激活次数"
- **原因**: 用户的激活次数配额已耗尽。
- **解决**: 管理员在 NavManage 后台编辑用户，增加 `max_activations` 数量。

## 五、API 接口说明 (参考)

| 接口路径 | 方法 | 说明 |
|---------|------|------|
| `/api/activation/activate` | POST | 激活授权 (Client -> Server) |
| `/api/activation/request-migrate` | POST | 申请迁移/新码 (Client -> Server) |
| `/api/licenses/validate` | POST | 在线验证授权状态 (Client -> Server) |
