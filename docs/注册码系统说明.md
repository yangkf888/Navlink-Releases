# NavLink 注册码系统说明

## 一、系统架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                        注册码生命周期                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① 生成密钥对              ② 签发 License           ③ 激活验证    │
│  ┌───────────────┐        ┌───────────────┐       ┌───────────┐ │
│  │generate-keys.js│  →    │sign-license.js│   →   │用户激活   │ │
│  │   (一次性)      │        │  (每用户一次)  │       │LicenseService│
│  └───────────────┘        └───────────────┘       └───────────┘ │
│        ↓                         ↓                      ↓       │
│   私钥 + 公钥            License Key (Base64)     验证签名       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 二、核心文件说明

| 文件 | 用途 | 执行时机 |
|-----|------|---------|
| `scripts/generate-keys.js` | 生成 ECDSA 密钥对 | **仅首次部署执行一次** |
| `scripts/sign-license.js` | 为用户签发 License Key | **每个用户执行一次** |
| `server/services/LicenseService.js` | 验证 License 有效性 | 服务器运行时 |
| `server/middleware/license.js` | 拦截未授权请求 | API 请求时 |
| `data/instance.id` | 机器唯一标识 (UUID) | 自动生成 |
| `data/license.key` | 用户激活的 License | 用户激活后存储 |

## 三、操作步骤

### 步骤 1：生成密钥对（仅一次）

```bash
node scripts/generate-keys.js
```

**输出示例：**
```

---------------------------------------------------

f94d69332530e475ed8e61652afdf60b35e0300c269983ab8cf74397965b9c47

🔒 公钥 (请填入 LicenseService.js):
04862015b7e5f1721724aa5c31794a16faec7b86399a32e5c97a5a34f0343d2039...
```

**操作：**
1. 将**公钥**复制到 `server/services/LicenseService.js` 第 20 行
2. 将**私钥**保存到安全位置，并填入 `scripts/sign-license.js` 第 18 行

---

### 步骤 2：获取用户的 Instance ID

用户部署后，需要获取其机器码：

**方法 A：用户从后台管理页面复制**
- 访问 `/admin/system` → 系统激活 → 显示 Instance ID

**方法 B：直接读取文件**
```bash
cat data/instance.id
# 输出: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

### 步骤 3：为用户签发 License

```bash
node scripts/sign-license.js user@example.com --instance=a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

**输出示例：**
```
✅ License Key 生成成功:
---------------------------------------------------
eyJpbnN0YW5jZUlkIjoiYTFiMmMzZDQtZTVmNi03ODkwLWFiY2QtZWYxMjM0NTY3ODkwIiwi...
```

**将此 License Key 发送给用户。**

---

### 步骤 4：用户激活

用户在后台管理页面输入 License Key 激活：
1. 访问 `/admin/system` → 系统激活
2. 粘贴 License Key
3. 点击激活

## 四、技术原理

### 加密算法
- **算法**: ECDSA (椭圆曲线数字签名算法)
- **曲线**: secp256k1 (比特币同款)
- **哈希**: SHA-256

### License 数据结构

```json
{
  "instanceId": "a1b2c3d4-...",      // 绑定的机器码
  "issuedTo": "user@example.com",   // 授权给谁
  "expiresAt": "2099-12-31T23:59:59Z", // 过期时间
  "features": ["all"],               // 功能权限
  "signature": "304402..."          // 数字签名
}
```

### 签名逻辑

```javascript
// 签发时 (sign-license.js)
const signString = `${instanceId}|${issuedTo}|${expiresAt}`;
const msgHash = sha256(signString);
const signature = privateKey.sign(msgHash);

// 验证时 (LicenseService.js)
const signString = `${instanceId}|${issuedTo}|${expiresAt}`;
const msgHash = sha256(signString);
const isValid = publicKey.verify(msgHash, signature);
```

## 五、安全机制

| 机制 | 说明 |
|-----|------|
| **机器绑定** | License 与 Instance ID 绑定，换机器需要重新签发 |
| **签名验证** | 只有拥有私钥才能签发有效 License |
| **过期检查** | 支持设置授权有效期 |
| **篡改防护** | 修改任何字段会导致签名验证失败 |

## 六、注意事项

### ⚠️ 私钥安全
- **绝对不能**将私钥提交到 Git 仓库
- 私钥泄露 = 任何人都能签发 License
- 建议将私钥存储在环境变量或密钥管理服务中

### ⚠️ 公钥更换
- 更换公钥后，**所有现有 License 都会失效**
- 需要为所有用户重新签发

### ⚠️ Instance ID
- `data/instance.id` 是每个部署实例的唯一标识
- 如果删除此文件，会生成新 ID，原 License 失效
- Docker 部署时需要持久化 `data/` 目录

### ⚠️ 时钟问题
- 过期检查依赖系统时间
- 如果用户调整系统时间，可能绕过过期检查
- 可考虑增加在线验证机制

## 八、常见问题

### Q: License 验证失败怎么办？

1. **检查 Instance ID 是否匹配**
   - 用户提供的 Instance ID 是否正确
   - `data/instance.id` 文件是否被重新生成

2. **检查 License 是否过期**
   - 默认设置为 2099 年，一般不会过期

3. **检查公钥是否正确**
   - `LicenseService.js` 中的公钥是否与签发时使用的私钥配对

### Q: 如何修改授权有效期？

编辑 `scripts/sign-license.js` 第 29 行：
```javascript
expiresAt: '2025-12-31T23:59:59Z', // 修改为具体日期
```

### Q: 如何撤销某个用户的授权？

目前没有实现撤销机制。可选方案：
1. 更换密钥对（影响所有用户）
2. 实现在线验证 + 黑名单机制
