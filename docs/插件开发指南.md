## 1. 概述

NavLink 是一个可扩展的导航管理平台，支持通过插件机制扩展功能。本指南将帮助你：

- 理解 NavLink 插件架构
- 学习如何创建新插件
- 使用共享组件和上下文
- 实现前端界面和后端 API
- 集成导航栏和侧边栏

### 1.1 插件能做什么？

- ✅ 提供独立的功能模块（如 Docker 管理、订阅管理、VPS 监控等）
- ✅ 拥有独立的前端页面和后端 API
- ✅ **侧边栏完全自治**：插件自行管理侧边栏逻辑、样式和移动端抽屉
- ✅ **主题自适应**：支持独立的暗黑模式/明亮模式切换
- ✅ 使用共享组件和样式，通过 Iframe 隔离保持主应用稳定性
- ✅ 独立的数据存储和业务逻辑

## 2. 插件架构

### 2.1 目录结构

```
plugins/
├── your-plugin/
│   ├── manifest.json          # 插件配置文件
│   ├── backend-nodejs/        # 后端代码
│   │   ├── server.js          # 后端入口文件
│   │   ├── routes/            # API 路由
│   │   ├── models/            # 数据模型
│   │   └── utils/             # 工具函数
│   ├── frontend/              # 前端代码
│   │   ├── src/
│   │   │   ├── App.tsx        # 前端入口组件
│   │   │   ├── components/    # 组件目录
│   │   │   ├── hooks/         # 自定义 Hooks
│   │   │   ├── types/         # TypeScript 类型定义
│   │   │   └── utils/         # 工具函数
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── your-plugin.db         # SQLite 数据库（可选）
```

### 2.2 插件生命周期

1. **注册阶段**：主应用扫描 `plugins/` 目录，读取 `manifest.json`
2. **启动阶段**：启动插件的 Node.js 后端服务（in-process模式）
3. **路由注册**：
   - 用户路由：`/apps/{plugin-id}` - 显示主应用布局（导航栏+侧边栏）
   - 内容路由：`/plugin-content/{plugin-id}` - iframe加载插件内容
4. **运行阶段**：插件在iframe中运行，通过postMessage与主应用通信
5. **卸载阶段**：停止后端服务，清理资源

## 3. 快速开始

### 3.1 创建插件配置文件

在 `plugins/your-plugin/` 目录下创建 `manifest.json`：

```json
{
  "id": "your-plugin",
  "name": "Your Plugin Name",
  "description": "Plugin description",
  "version": "1.0.0",
  "type": "node",
  "entry": "backend-nodejs/server.js",
  "healthCheck": "/api/health",
  "frontend": "frontend/dist"
}
```

**字段说明**：
- `id`：插件唯一标识符（小写字母、数字、连字符）
- `name`：插件显示名称
- `description`：插件描述
- `version`：版本号（遵循 SemVer）
- `type`：插件类型（目前仅支持 `node`）
- `entry`：后端入口文件路径（相对于插件目录）
- `healthCheck`：健康检查 API 路径
- `frontend`：前端构建产物目录路径

### 3.2 创建后端服务

在 `backend-nodejs/server.js` 中：

```javascript
const express = require('express');
const cors = require('cors');
const path = require('path');

const app = express();
const PORT = process.env.PORT || 3100; // 插件需使用独立端口

// 中间件
app.use(cors());
app.use(express.json());

// 健康检查
app.get('/api/health', (req, res) => {
  res.json({ status: 'healthy', plugin: 'your-plugin' });
});

// API 路由
app.get('/api/data', (req, res) => {
  res.json({ message: 'Hello from plugin!' });
});

// 启动服务器
app.listen(PORT, () => {
  console.log(`[Your Plugin] Backend running on http://localhost:${PORT}`);
});
```

### 3.3 创建前端应用

在 `frontend/src/App.tsx` 中：

```typescript
import React, { useState, useEffect } from 'react';

function YourPluginApp() {
  const [activeView, setActiveView] = useState('home');
  const [theme, setTheme] = useState(() => localStorage.getItem('your_plugin_theme') || 'light');

  // 1. 主题应用
  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('your_plugin_theme', theme);
  }, [theme]);

  // 2. 自治侧边栏配置发送 (告诉主应用不要显示其旧版侧边栏)
  useEffect(() => {
    const isInIframe = window.parent !== window;
    if (!isInIframe) return;

    window.parent.postMessage({
      type: 'PLUGIN_SET_SIDEBAR',
      payload: { items: [], activeId: '' } // 发送空 items 以激活自治模式
    }, '*');

    // 请求在桌面端显示主应用导航栏，移动端隐藏
    window.parent.postMessage({
      type: 'PLUGIN_REQUEST_HIDE_HEADER',
      payload: { hideHeader: false }
    }, '*');
  }, []);

  return (
    <div className="flex h-screen bg-[var(--theme-bg)]">
      {/* 插件内部渲染自己的 Sidebar */}
      <aside className="w-64 border-r border-gray-200 hidden lg:block">
         {/* ... 菜单项 ... */}
         <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
            切换模式
         </button>
      </aside>
      
      <main className="flex-1 overflow-auto p-6">
        <h1 className="text-2xl font-bold">欢迎使用自治插件！</h1>
      </main>
    </div>
  );
}

export default YourPluginApp;
```

### 4.4 暗黑模式接入

NavLink 插件系统支持基于 CSS 变量的主题方案。

#### 1. 定义变量 (index.css)
```css
:root, [data-theme="light"] {
  --theme-bg: #f8f9fa;
  --theme-text: #1f2937;
  --theme-border: #e5e7eb;
}

[data-theme="dark"] {
  --theme-bg: #0b0f17;
  --theme-text: #f0f6fc;
  --theme-border: rgba(255, 255, 255, 0.08);
}
```

#### 2. 实现切换逻辑
在 `App.tsx` 或 `Layout` 组件中设置属性：
```typescript
document.documentElement.setAttribute('data-theme', 'dark');
```

#### 3. 覆盖 Tailwind 类
由于插件中大量使用 Tailwind 颜色类（如 `bg-white`），在暗黑模式下需要使用 `!important` 强制覆盖：
```css
[data-theme="dark"] .bg-white {
  background-color: #161b22 !important;
}
[data-theme="dark"] .text-gray-900 {
  color: #f0f6fc !important;
}
```

## 5. 后端开发

### 5.1 API 设计原则

1. **路径前缀**：所有 API 路径应以 `/api` 开头
2. **RESTful 风格**：遵循 REST API 设计规范
3. **错误处理**：统一的错误响应格式
4. **认证集成**：可选择性地集成主应用的认证机制

### 5.1.5 前端API调用规范 ⚠️ **重要**

**问题**：插件在iframe中运行时，相对路径（如`./api/data`）会错误解析。

**解决方案**：使用API工具函数统一管理路径

#### 创建API工具

在 `frontend/src/utils/api.ts` 中：

```typescript
/**
 * 插件API工具 - 将相对路径转换为绝对路径
 */
const PLUGIN_ID = 'your-plugin'; // 改为你的插件ID

const API_BASE = `/api/plugins/${PLUGIN_ID}`;

/**
 * 将相对路径转换为绝对API路径
 * @param path API路径，如 'api/items' 或 './api/items'
 * @returns 绝对路径，如 '/api/plugins/your-plugin/api/items'
 */
export const apiUrl = (path: string): string => {
  const cleanPath = path.replace(/^\.?\//, '');
  return `${API_BASE}/${cleanPath}`;
};

/**
 * 带认证的fetch封装（可选）
 */
export const apiFetch = async (path: string, options: RequestInit = {}) => {
  const token = localStorage.getItem('auth_token');
  return fetch(apiUrl(path), {
    ...options,
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json',
      ...options.headers,
    }
  });
};
```

#### 在组件中使用

```typescript
import { apiUrl } from '../utils/api';

// ❌ 错误：使用相对路径
fetch('./api/items')

// ✅ 正确：使用apiUrl包裹
fetch(apiUrl('api/items'))

// ✅ 完整示例
async function loadItems() {
  const response = await fetch(apiUrl('api/items'), {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  const data = await response.json();
  return data;
}
```

**原理**：
- 插件iframe从 `/plugin-content/your-plugin/` 加载
- 相对路径 `./api/items` 会解析为 `/plugin-content/your-plugin/api/items` ❌
- 正确路径应该是 `/api/plugins/your-plugin/api/items` ✅
- `apiUrl()` 自动添加正确的前缀

### 5.4 用户认证与上下文

#### 5.4.1 认证机制说明

NavLink 插件系统采用**基于HTTP Headers的用户上下文传递**机制，而非要求插件验证JWT token。

**核心原则**：
- ✅ 插件后端**无需**验证JWT token
- ✅ 主应用通过HTTP Headers传递用户信息给插件
- ✅ 前端的`auth_token`是**可选的**，主要用于主应用认证
- ✅ 插件可以直接信任主应用传递的用户上下文

#### 5.4.2 用户上下文Headers

主应用会自动注入以下HTTP headers到每个插件请求：

```http
X-Nav-User-Id: user_1001          # 用户ID
X-Nav-Tenant-Id: default           # 租户ID（多租户支持）
X-Nav-User-Role: user              # 用户角色（admin/user）
X-Nav-Username: default-user       # 用户名
```

**使用示例**：

```javascript
// routes/items.js
router.get('/', async (req, res) => {
  // 从headers获取用户信息
  const userId = req.headers['x-nav-user-id'];
  const tenantId = req.headers['x-nav-tenant-id'];
  const userRole = req.headers['x-nav-user-role'];

  console.log(`[Items] User ${userId} from tenant ${tenantId} is accessing items`);

  try {
    // 基于租户ID查询数据（实现租户隔离）
    const items = await db.getItemsByTenant(tenantId);
    res.json({ success: true, data: items });
  } catch (error) {
    res.status(500).json({ success: false, error: error.message });
  }
});

// 创建项目时记录用户信息
router.post('/', async (req, res) => {
  const userId = req.headers['x-nav-user-id'];
  const tenantId = req.headers['x-nav-tenant-id'];

  try {
    const newItem = await db.createItem({
      ...req.body,
      user_id: userId,      // 记录创建者
      tenant_id: tenantId   // 租户隔离
    });
    res.json({ success: true, data: newItem });
  } catch (error) {
    res.status(400).json({ success: false, error: error.message });
  }
});
```

#### 5.4.3 租户隔离最佳实践

在多租户环境中，确保数据隔离非常重要：

```javascript
// database/dao/ItemDAO.js
class ItemDAO {
  // 获取租户的所有项目
  getItemsByTenant(tenantId) {
    return new Promise((resolve, reject) => {
      this.db.all(
        'SELECT * FROM items WHERE tenant_id = ?',
        [tenantId],
        (err, rows) => {
          if (err) reject(err);
          else resolve(rows);
        }
      );
    });
  }
  
  // 创建项目时自动关联租户
  createItem(data) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'INSERT INTO items (name, type, tenant_id, user_id) VALUES (?, ?, ?, ?)',
        [data.name, data.type, data.tenant_id, data.user_id],
        function(err) {
          if (err) reject(err);
          else resolve({ id: this.lastID, ...data });
        }
      );
    });
  }
}
```

#### 5.4.4 前端Token的作用

前端使用`localStorage.getItem('auth_token')`主要用于：
- ✅ 主应用的登录状态管理
- ✅ ConfigContext中的`isAuthenticated`状态
- ⚠️ **插件后端不依赖此token**

**前端API调用示例**：

```typescript
// hooks/useItems.ts
export function useItems() {
  const loadItems = async () => {
    try {
      // 可选：发送auth_token（主应用会验证）
      const token = localStorage.getItem('auth_token');
      const headers: HeadersInit = {
        'Content-Type': 'application/json'
      };
      if (token) {
        headers['Authorization'] = `Bearer ${token}`;
      }

      const response = await fetch('/api/plugins/your-plugin/api/items', {
        headers
      });

      if (!response.ok) throw new Error('Failed to load items');
      const data = await response.json();
      return data;
    } catch (error) {
      console.error('[Items] Load failed:', error);
      throw error;
    }
  };

  return { loadItems };
}
```

> **重要提示**：
> - 即使前端不发送`auth_token`，主应用也会提供默认用户上下文
> - 插件后端只需关注`X-Nav-*` headers，无需处理JWT验证
> - 认证和授权由主应用统一管理

#### 5.4.5 权限检查（可选）

```javascript
});
```

## 7. Context 使用指南

### 7.1 ConfigContext

提供全局配置和认证状态：

```typescript
import { useConfig } from '@/shared/context/ConfigContext';

function YourComponent() {
  const { config, isLoaded, isAuthenticated, login, logout } = useConfig();

  if (!isLoaded) {
    return <div>加载配置中...</div>;
  }

  return (
    <div>
      <h1>{config.siteName}</h1>
      <p>主题色：{config.theme?.navbarBgColor}</p>
      {isAuthenticated ? (
        <button onClick={logout}>退出登录</button>
      ) : (
        <button onClick={() => login('username', 'password')}>登录</button>
      )}
    </div>
  );
}
```

**可用字段**：
- `config: SiteConfig` - 站点配置对象
- `isLoaded: boolean` - 配置是否已加载
- `isAuthenticated: boolean` - 用户是否已认证
- `login(username, password): Promise<void>` - 登录方法
- `logout(): void` - 登出方法

### 7.2 LayoutContext

## 8. 侧边栏与布局自治 (NEW)

### 8.1 架构原理

**重要更新 (v2.1)**：插件现在采用**完全自治架构**。这意味着：
- ✅ **侧边栏不在主应用渲染**：插件在 iframe 内部渲染自己的 `Sidebar`。
- ✅ **移动端独立管理**：插件需要渲染自己的移动端顶部栏 (Hamburger Menu) 和抽屉。
- ✅ **主应用静默模式**：主应用仅作为一个透明的 iframe 容器，不干扰插件内部 UI。

```typescript
            {activeView === 'dashboard' && <Dashboard />}
         </main>

      {/* 移动端抽屉 (Drawer) */}
      {mobileOpen && (
         <div className="fixed inset-0 z-50 lg:hidden">
            <div className="absolute inset-0 bg-black/40" onClick={() => setMobileOpen(false)} />
```

## 10. 数据持久化

### 10.1 使用 SQLite

推荐使用 SQLite 作为插件的数据库：

```javascript
const sqlite3 = require('sqlite3').verbose();
const path = require('path');

// 数据库文件位于插件根目录
const DB_PATH = path.join(__dirname, '..', 'your-plugin.db');

class Database {
  constructor() {
    this.db = new sqlite3.Database(DB_PATH, (err) => {
      if (err) {
        console.error('[DB] Error opening database:', err);
      } else {
        console.log('[DB] Database connected');
        this.init();
      }
    });
  }

  init() {
    this.db.run(`
      CREATE TABLE IF NOT EXISTS items (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        type TEXT NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
      )
    `);
  }

  // 查询所有
  getAllItems() {
    return new Promise((resolve, reject) => {
      this.db.all('SELECT * FROM items', [], (err, rows) => {
        if (err) reject(err);
        else resolve(rows);
      });
    });
  }

  // 创建
  createItem(name, type) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'INSERT INTO items (name, type) VALUES (?, ?)',
        [name, type],
        function (err) {
          if (err) reject(err);
          else resolve({ id: this.lastID, name, type });
        }
      );
    });
  }

  // 更新
  updateItem(id, data) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'UPDATE items SET name = ?, type = ? WHERE id = ?',
        [data.name, data.type, id],
        (err) => {
          if (err) reject(err);
          else resolve({ id, ...data });
        }
      );
    });
  }

  // 删除
  deleteItem(id) {
    return new Promise((resolve, reject) => {
      this.db.run('DELETE FROM items WHERE id = ?', [id], (err) => {
        if (err) reject(err);
        else resolve({ id });
      });
    });
  }
}

module.exports = new Database();
```
