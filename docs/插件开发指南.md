## 1. 概述

NavLink 是一个可扩展的导航管理平台，支持通过插件机制扩展功能。本指南将帮助你：

- 理解 NavLink 插件架构
- 学习如何创建新插件
- 使用共享组件和上下文
- 实现前端界面和后端 API
- 集成导航栏和侧边栏

### 1.1 插件能做什么？

- ✅ 提供独立的功能模块（如 Docker 管理、订阅管理、VPS 监控等）
- ✅ 拥有独立的前端页面和后端 API
- ✅ 复用主应用的导航栏、侧边栏、认证等基础设施
- ✅ 使用共享组件和样式，保持 UI 一致性
- ✅ 独立的数据存储和业务逻辑

## 2. 插件架构

### 2.1 目录结构

```
plugins/
├── your-plugin/
│   ├── manifest.json          # 插件配置文件
│   ├── backend-nodejs/        # 后端代码
│   │   ├── server.js          # 后端入口文件
│   │   ├── routes/            # API 路由
│   │   ├── models/            # 数据模型
│   │   └── utils/             # 工具函数
│   ├── frontend/              # 前端代码
│   │   ├── src/
│   │   │   ├── App.tsx        # 前端入口组件
│   │   │   ├── components/    # 组件目录
│   │   │   ├── hooks/         # 自定义 Hooks
│   │   │   ├── types/         # TypeScript 类型定义
│   │   │   └── utils/         # 工具函数
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── your-plugin.db         # SQLite 数据库（可选）
```

### 2.2 插件生命周期

1. **注册阶段**：主应用扫描 `plugins/` 目录，读取 `manifest.json`
2. **启动阶段**：启动插件的 Node.js 后端服务
3. **路由注册**：前端路由自动注册到 `/apps/{plugin-id}/*`
4. **运行阶段**：插件独立运行，通过 API 与主应用通信
5. **卸载阶段**：停止后端服务，清理资源

## 3. 快速开始

### 3.1 创建插件配置文件

在 `plugins/your-plugin/` 目录下创建 `manifest.json`：

```json
{
  "id": "your-plugin",
  "name": "Your Plugin Name",
  "description": "Plugin description",
  "version": "1.0.0",
  "type": "node",
  "entry": "backend-nodejs/server.js",
  "healthCheck": "/api/health",
  "frontend": "frontend/dist"
}
```

**字段说明**：
- `id`：插件唯一标识符（小写字母、数字、连字符）
- `name`：插件显示名称
- `description`：插件描述
- `version`：版本号（遵循 SemVer）
- `type`：插件类型（目前仅支持 `node`）
- `entry`：后端入口文件路径（相对于插件目录）
- `healthCheck`：健康检查 API 路径
- `frontend`：前端构建产物目录路径

### 3.2 创建后端服务

在 `backend-nodejs/server.js` 中：

```javascript
const express = require('express');
const cors = require('cors');
const path = require('path');

const app = express();
const PORT = process.env.PORT || 3100; // 插件需使用独立端口

// 中间件
app.use(cors());
app.use(express.json());

// 健康检查
app.get('/api/health', (req, res) => {
  res.json({ status: 'healthy', plugin: 'your-plugin' });
});

// API 路由
app.get('/api/data', (req, res) => {
  res.json({ message: 'Hello from plugin!' });
});

// 启动服务器
app.listen(PORT, () => {
  console.log(`[Your Plugin] Backend running on http://localhost:${PORT}`);
});
```

### 3.3 创建前端应用

在 `frontend/src/App.tsx` 中：

```typescript
import React, { useState } from 'react';
import { useConfig } from '@/shared/context/ConfigContext';
import { useLayout } from '@/shared/context/LayoutContext';

function YourPluginApp() {
  const { config, isAuthenticated } = useConfig();
  const { setSidebarContent, collapsed } = useLayout();

    setSidebarContent(

        </button>
      </div>

    return () => setSidebarContent(null);
  }, [setSidebarContent, collapsed]);

  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold">欢迎使用插件！</h1>
    </div>
  );
}

export default YourPluginApp;
```

### 5.4 用户认证与上下文

#### 5.4.1 认证机制说明

NavLink 插件系统采用**基于HTTP Headers的用户上下文传递**机制，而非要求插件验证JWT token。

**核心原则**：
- ✅ 插件后端**无需**验证JWT token
- ✅ 主应用通过HTTP Headers传递用户信息给插件
- ✅ 前端的`auth_token`是**可选的**，主要用于主应用认证
- ✅ 插件可以直接信任主应用传递的用户上下文

#### 5.4.2 用户上下文Headers

主应用会自动注入以下HTTP headers到每个插件请求：

```http
X-Nav-User-Id: user_1001          # 用户ID
X-Nav-Tenant-Id: default           # 租户ID（多租户支持）
X-Nav-User-Role: user              # 用户角色（admin/user）
X-Nav-Username: default-user       # 用户名
```

**使用示例**：

```javascript
// routes/items.js
router.get('/', async (req, res) => {
  // 从headers获取用户信息
  const userId = req.headers['x-nav-user-id'];
  const tenantId = req.headers['x-nav-tenant-id'];
  const userRole = req.headers['x-nav-user-role'];

  console.log(`[Items] User ${userId} from tenant ${tenantId} is accessing items`);

  try {
    // 基于租户ID查询数据（实现租户隔离）
    const items = await db.getItemsByTenant(tenantId);
    res.json({ success: true, data: items });
  } catch (error) {
    res.status(500).json({ success: false, error: error.message });
  }
});

// 创建项目时记录用户信息
router.post('/', async (req, res) => {
  const userId = req.headers['x-nav-user-id'];
  const tenantId = req.headers['x-nav-tenant-id'];

  try {
    const newItem = await db.createItem({
      ...req.body,
      user_id: userId,      // 记录创建者
      tenant_id: tenantId   // 租户隔离
    });
    res.json({ success: true, data: newItem });
  } catch (error) {
    res.status(400).json({ success: false, error: error.message });
  }
});
```

#### 5.4.3 租户隔离最佳实践

在多租户环境中，确保数据隔离非常重要：

```javascript
// database/dao/ItemDAO.js
class ItemDAO {
  // 获取租户的所有项目
  getItemsByTenant(tenantId) {
    return new Promise((resolve, reject) => {
      this.db.all(
        'SELECT * FROM items WHERE tenant_id = ?',
        [tenantId],
        (err, rows) => {
          if (err) reject(err);
          else resolve(rows);
        }
      );
    });
  }
  
  // 创建项目时自动关联租户
  createItem(data) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'INSERT INTO items (name, type, tenant_id, user_id) VALUES (?, ?, ?, ?)',
        [data.name, data.type, data.tenant_id, data.user_id],
        function(err) {
          if (err) reject(err);
          else resolve({ id: this.lastID, ...data });
        }
      );
    });
  }
}
```

#### 5.4.4 前端Token的作用

前端使用`localStorage.getItem('auth_token')`主要用于：
- ✅ 主应用的登录状态管理
- ✅ ConfigContext中的`isAuthenticated`状态
- ⚠️ **插件后端不依赖此token**

**前端API调用示例**：

```typescript
// hooks/useItems.ts
export function useItems() {
  const loadItems = async () => {
    try {
      // 可选：发送auth_token（主应用会验证）
      const token = localStorage.getItem('auth_token');
      const headers: HeadersInit = {
        'Content-Type': 'application/json'
      };
      if (token) {
        headers['Authorization'] = `Bearer ${token}`;
      }

      const response = await fetch('/api/plugins/your-plugin/api/items', {
        headers
      });

      if (!response.ok) throw new Error('Failed to load items');
      const data = await response.json();
      return data;
    } catch (error) {
      console.error('[Items] Load failed:', error);
      throw error;
    }
  };

  return { loadItems };
}
```

> **重要提示**：
> - 即使前端不发送`auth_token`，主应用也会提供默认用户上下文
> - 插件后端只需关注`X-Nav-*` headers，无需处理JWT验证
> - 认证和授权由主应用统一管理

#### 5.4.5 权限检查（可选）

```javascript
  }
});
```

## 8. 侧边栏集成（iframe + postMessage 架构）

### 8.1 架构概述

**重要变更（v2.0）**：插件现在运行在 iframe 中，通过 `postMessage` 与主应用通信，实现：
- ✅ **统一 UI**：所有插件共享主应用的 TopNavbar
- ✅ **动态 Sidebar**：每个插件自定义侧边栏内容
- ✅ **完全隔离**：插件代码独立运行，互不干扰
- ✅ **动态安装**：无需修改主应用代码，即可安装新插件

**插件职责**：
- 插件只需渲染**内容区域**
- **不渲染** TopNavbar 和 Sidebar（由主应用提供）
- 通过 `postMessage` 发送侧边栏配置
- 监听主应用的点击事件

**主应用职责**：
- 提供 TopNavbar
- 渲染动态 Sidebar（根据插件配置）
- 转发用户点击事件给插件

### 8.2 postMessage 通信协议

#### 8.2.1 发送 Sidebar 配置（插件 → 主应用）

插件在组件挂载时和状态变化时发送侧边栏配置：

```typescript
// 消息类型
interface PluginSetSidebarMessage {
  type: 'PLUGIN_SET_SIDEBAR';
  payload: {
    title?: string;        // 侧边栏标题（可选）
    items: SidebarItem[];  // 菜单项列表
    activeId: string;      // 当前激活的菜单项ID
  };
}

interface SidebarItem {
  id: string;              // 菜单项唯一标识
  label: string;           // 显示文字
  icon: string;            // Font Awesome 图标类名
}
```

**代码示例**：

```typescript
import { useState, useEffect } from 'react';

function YourPluginApp() {
  const [activeView, setActiveView] = useState('dashboard');

  // 🔑 发送 Sidebar 配置
  useEffect(() => {
    const isInIframe = window.parent !== window;
    if (!isInIframe) return;

    const sidebarConfig = {
      title: '你的插件名称',
      items: [
        { id: 'dashboard', label: '仪表盘', icon: 'fas fa-home' },
        { id: 'list', label: '列表', icon: 'fas fa-list' },
        { id: 'settings', label: '设置', icon: 'fas fa-cog' }
      ],
      activeId: activeView
    };

    window.parent.postMessage({
      type: 'PLUGIN_SET_SIDEBAR',
      payload: sidebarConfig
    }, '*');
  }, [activeView]);

  // 🔑 组件挂载时立即发送初始配置
  useEffect(() => {
    const isInIframe = window.parent !== window;
    if (!isInIframe) return;

    window.parent.postMessage({
      type: 'PLUGIN_SET_SIDEBAR',
      payload: {
        title: '你的插件名称',
        items: [
          { id: 'dashboard', label: '仪表盘', icon: 'fas fa-home' },
          { id: 'list', label: '列表', icon: 'fas fa-list' },
          { id: 'settings', label: '设置', icon: 'fas fa-cog' }
        ],
        activeId: 'dashboard'
      }
    }, '*');
  }, []);

  return (
    <div className="p-6">
      {/* 只渲染内容区域，不需要 TopNavbar 和 Sidebar */}
      {activeView === 'dashboard' && <DashboardView />}
      {activeView === 'list' && <ListView />}
      {activeView === 'settings' && <SettingsView />}
    </div>
  );
}
```

#### 8.2.2 监听点击事件（主应用 → 插件）

当用户点击侧边栏菜单时，主应用会发送点击事件：

```typescript
// 消息类型
interface SidebarItemClickedMessage {
  type: 'SIDEBAR_ITEM_CLICKED';
  payload: {
    itemId: string;  // 被点击的菜单项ID
  };
}
```

```typescript
      }
```

### 8.5 最佳实践

#### ✅ 推荐做法

1. **始终检查 iframe 环境**
   ```typescript
   const isInIframe = window.parent !== window;
   if (!isInIframe) return;
   ```

2. **添加调试日志**
   ```typescript
   console.log('[Your Plugin] Sending sidebar config:', sidebarConfig);
   console.log('[Your Plugin] Received click event:', itemId);
   ```

3. **立即发送初始配置**
   - 在组件挂载时发送一次（不等待数据加载）
   - 在状态变化时更新配置

4. **清理事件监听器**
   ```typescript
   useEffect(() => {
     window.addEventListener('message', handleMessage);
     return () => window.removeEventListener('message', handleMessage);
   }, []);
   ```

#### ❌ 避免的问题

1. **不要忘记检查 iframe 环境**
   - 在开发模式下可能直接访问插件URL，不在iframe中
   - 必须检查 `window.parent !== window`

2. **不要过度发送消息**
   - 不要在每次渲染时都发送消息
   - 使用 `useEffect` 的依赖数组控制发送时机

3. **不要阻塞UI**
   - postMessage 是异步的
   - 主应用会立即响应并渲染侧边栏

4. **不要使用旧的 `useLayout` 方式**
   - ❌ `const { setSidebarContent } = useLayout();`
   - ✅ `window.parent.postMessage({ type: 'PLUGIN_SET_SIDEBAR', ... })`

## 10. 数据持久化

### 10.1 使用 SQLite

推荐使用 SQLite 作为插件的数据库：

```javascript
const sqlite3 = require('sqlite3').verbose();
const path = require('path');

// 数据库文件位于插件根目录
const DB_PATH = path.join(__dirname, '..', 'your-plugin.db');

class Database {
  constructor() {
    this.db = new sqlite3.Database(DB_PATH, (err) => {
      if (err) {
        console.error('[DB] Error opening database:', err);
      } else {
        console.log('[DB] Database connected');
        this.init();
      }
    });
  }

  init() {
    this.db.run(`
      CREATE TABLE IF NOT EXISTS items (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        type TEXT NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
      )
    `);
  }

  // 查询所有
  getAllItems() {
    return new Promise((resolve, reject) => {
      this.db.all('SELECT * FROM items', [], (err, rows) => {
        if (err) reject(err);
        else resolve(rows);
      });
    });
  }

  // 创建
  createItem(name, type) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'INSERT INTO items (name, type) VALUES (?, ?)',
        [name, type],
        function (err) {
          if (err) reject(err);
          else resolve({ id: this.lastID, name, type });
        }
      );
    });
  }

  // 更新
  updateItem(id, data) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'UPDATE items SET name = ?, type = ? WHERE id = ?',
        [data.name, data.type, id],
        (err) => {
          if (err) reject(err);
          else resolve({ id, ...data });
        }
      );
    });
  }

  // 删除
  deleteItem(id) {
    return new Promise((resolve, reject) => {
      this.db.run('DELETE FROM items WHERE id = ?', [id], (err) => {
        if (err) reject(err);
        else resolve({ id });
      });
    });
  }
}

module.exports = new Database();
```
