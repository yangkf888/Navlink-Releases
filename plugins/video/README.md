# NavLink Video Plugin

## 视频转码功能说明 (Transcoding)

为了支持 STRM 格式（通常包含 HEVC/H.265 编码）在浏览器中的播放，本插件集成了 FFmpeg 转码服务。

### FFmpeg 安装方式

插件支持两种方式调用 FFmpeg，按照以下优先级自动检测：

#### 1. 便携模式 (推荐容器/无法修改系统环境时使用)
插件会自动检测是否在数据目录中包含 `ffmpeg` 可执行文件。
- **路径**: `data/bin/ffmpeg` (相对于插件根目录的 `../../data/bin/ffmpeg`)
- **适用场景**: Docker 容器部署、无 Root 权限环境

**安装步骤**:
1. 下载对应系统的 FFmpeg 静态编译文件 (如 amd64 Linux 版)。
2. 将可执行文件重命名为 `ffmpeg` (无后缀)。
3. 将其上传到 NavLink 数据目录下的 `bin` 文件夹中。
   - 完整路径示例: `/navlink/data/bin/ffmpeg`
4. 确保文件具有可执行权限 (`chmod +x ffmpeg`)。
5. 重启插件或 NavLink。

#### 2. 系统环境变量模式
如果未检测到便携版文件，插件将尝试使用系统 PATH 环境变量中的 `ffmpeg` 命令。
- **适用场景**: 本地开发、宿主机直接部署
- **安装**: 需确保宿主机已安装 FFmpeg (`apt install ffmpeg` 或 `brew install ffmpeg`)。

### 缓存清理
转码过程会产生临时切片文件，插件内置了自动清理机制：
- **实时清理**: 即使正在播放，也仅保留最新的 10 个分片。
- **会话结束**: 停止播放后立即删除所有临时文件。
- **定时任务**: 每分钟检查一次，自动回收超过 2 分钟无活动的僵尸会话。
