## 1. 概述

NavLink 是一个可扩展的导航管理平台，支持通过插件机制扩展功能。本指南将帮助你：

- 理解 NavLink 插件架构
- 学习如何创建新插件
- 使用共享组件和上下文
- 实现前端界面和后端 API
- 集成导航栏和侧边栏

### 1.1 插件能做什么？

- ✅ 提供独立的功能模块（如 Docker 管理、订阅管理、VPS 监控等）
- ✅ 拥有独立的前端页面和后端 API
- ✅ 复用主应用的导航栏、侧边栏、认证等基础设施
- ✅ 使用共享组件和样式，保持 UI 一致性
- ✅ 独立的数据存储和业务逻辑

## 2. 插件架构

### 2.1 目录结构

```
plugins/
├── your-plugin/
│   ├── manifest.json          # 插件配置文件
│   ├── backend-nodejs/        # 后端代码
│   │   ├── server.js          # 后端入口文件
│   │   ├── routes/            # API 路由
│   │   ├── models/            # 数据模型
│   │   └── utils/             # 工具函数
│   ├── frontend/              # 前端代码
│   │   ├── src/
│   │   │   ├── App.tsx        # 前端入口组件
│   │   │   ├── components/    # 组件目录
│   │   │   ├── hooks/         # 自定义 Hooks
│   │   │   ├── types/         # TypeScript 类型定义
│   │   │   └── utils/         # 工具函数
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── your-plugin.db         # SQLite 数据库（可选）
```

### 2.2 插件生命周期

1. **注册阶段**：主应用扫描 `plugins/` 目录，读取 `manifest.json`
2. **启动阶段**：启动插件的 Node.js 后端服务
3. **路由注册**：前端路由自动注册到 `/apps/{plugin-id}/*`
4. **运行阶段**：插件独立运行，通过 API 与主应用通信
5. **卸载阶段**：停止后端服务，清理资源

## 3. 快速开始

### 3.1 创建插件配置文件

在 `plugins/your-plugin/` 目录下创建 `manifest.json`：

```json
{
  "id": "your-plugin",
  "name": "Your Plugin Name",
  "description": "Plugin description",
  "version": "1.0.0",
  "type": "node",
  "entry": "backend-nodejs/server.js",
  "healthCheck": "/api/health",
  "frontend": "frontend/dist"
}
```

**字段说明**：
- `id`：插件唯一标识符（小写字母、数字、连字符）
- `name`：插件显示名称
- `description`：插件描述
- `version`：版本号（遵循 SemVer）
- `type`：插件类型（目前仅支持 `node`）
- `entry`：后端入口文件路径（相对于插件目录）
- `healthCheck`：健康检查 API 路径
- `frontend`：前端构建产物目录路径

### 3.2 创建后端服务

在 `backend-nodejs/server.js` 中：

```javascript
const express = require('express');
const cors = require('cors');
const path = require('path');

const app = express();
const PORT = process.env.PORT || 3100; // 插件需使用独立端口

// 中间件
app.use(cors());
app.use(express.json());

// 健康检查
app.get('/api/health', (req, res) => {
  res.json({ status: 'healthy', plugin: 'your-plugin' });
});

// API 路由
app.get('/api/data', (req, res) => {
  res.json({ message: 'Hello from plugin!' });
});

// 启动服务器
app.listen(PORT, () => {
  console.log(`[Your Plugin] Backend running on http://localhost:${PORT}`);
});
```

### 3.3 创建前端应用

在 `frontend/src/App.tsx` 中：

```typescript
import React, { useState } from 'react';
import { useConfig } from '@/shared/context/ConfigContext';
import { useLayout } from '@/shared/context/LayoutContext';

function YourPluginApp() {
  const { config, isAuthenticated } = useConfig();
  const { setSidebarContent, collapsed } = useLayout();

  // 注入侧边栏内容
  React.useEffect(() => {
    setSidebarContent(
      <div className="space-y-1 py-2">
        <button className="w-full flex items-center px-4 py-3 text-[14px] font-medium rounded-lg">
          <span className="w-6 text-center mr-2">🏠</span> 
          {!collapsed && "首页"}
        </button>
      </div>
    );
    return () => setSidebarContent(null);
  }, [setSidebarContent, collapsed]);

  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold">欢迎使用插件！</h1>
    </div>
  );
}

export default YourPluginApp;
```

### 4.2 依赖共享

插件前端代码可以导入主应用的共享模块（通过 `@/shared` 路径别名）：

```typescript
// 导入共享 Context
import { useConfig } from '@/shared/context/ConfigContext';
import { useLayout } from '@/shared/context/LayoutContext';

// 导入共享组件
import { Icon } from '@/shared/components/common/Icon';

// 导入共享 Hooks

```

**重要提示**：
- ✅ 使用共享组件和样式，保持 UI 一致性
- ✅ 使用共享 Context 访问全局配置和状态
- ❌ 不要在插件中创建自己的 `node_modules`（会导致 React 多实例问题）
- ❌ 不要覆盖主应用的全局样式

### 4.3 构建配置

插件前端使用 Vite 构建，确保 `vite.config.ts` 配置正确：

```typescript
import { defineConfig } from 'vite';

  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, '../../src'), // 指向主应用 src 目录
    },
  },
  build: {
    outDir: 'dist',
    emptyOutDir: true,
  },
});
```

## 5. 后端开发

### 5.1 API 设计原则

1. **路径前缀**：所有 API 路径应以 `/api` 开头
2. **RESTful 风格**：遵循 REST API 设计规范
3. **错误处理**：统一的错误响应格式
4. **认证集成**：可选择性地集成主应用的认证机制

### 5.2 路由组织

推荐使用 Express Router 组织路由：

```javascript
// routes/items.js
const express = require('express');
const router = express.Router();

// 获取列表
router.get('/', async (req, res) => {
  try {
    const items = await getItemsFromDB();
    res.json({ success: true, data: items });
  } catch (error) {
    res.status(500).json({ success: false, error: error.message });
  }
});

// 创建项目
router.post('/', async (req, res) => {
  try {
    const newItem = await createItem(req.body);
    res.json({ success: true, data: newItem });
  } catch (error) {
    res.status(400).json({ success: false, error: error.message });
  }
});

module.exports = router;
```

```javascript
// server.js
const itemsRouter = require('./routes/items');
app.use('/api/items', itemsRouter);
```

### 5.3 数据验证

使用中间件进行请求数据验证：

```javascript
const validateItem = (req, res, next) => {
  const { name, type } = req.body;
  
  if (!name || typeof name !== 'string') {
    return res.status(400).json({ 
      success: false, 
      error: 'Invalid name' 
    });
  }
  
  if (!['type1', 'type2'].includes(type)) {
    return res.status(400).json({ 
      success: false, 
      error: 'Invalid type' 
    });
  }
  
  next();
};

router.post('/', validateItem, async (req, res) => {
  // 处理逻辑
});
```

## 7. Context 使用指南

### 7.1 ConfigContext

提供全局配置和认证状态：

```typescript
import { useConfig } from '@/shared/context/ConfigContext';

function YourComponent() {
  const { config, isLoaded, isAuthenticated, login, logout } = useConfig();

  if (!isLoaded) {
    return <div>加载配置中...</div>;
  }

  return (
    <div>
      <h1>{config.siteName}</h1>
      <p>主题色：{config.theme?.navbarBgColor}</p>
      {isAuthenticated ? (
        <button onClick={logout}>退出登录</button>
      ) : (
        <button onClick={() => login('username', 'password')}>登录</button>
      )}
    </div>
  );
}
```

**可用字段**：
- `config: SiteConfig` - 站点配置对象
- `isLoaded: boolean` - 配置是否已加载
- `isAuthenticated: boolean` - 用户是否已认证
- `login(username, password): Promise<void>` - 登录方法
- `logout(): void` - 登出方法

### 7.2 LayoutContext

管理布局状态（侧边栏内容、收起状态等）：

```typescript
import { useLayout } from '@/shared/context/LayoutContext';

function YourComponent() {
  const { 
    sidebarContent,      // 当前侧边栏内容
    setSidebarContent,   // 设置侧边栏内容
    sidebarTitle,        // 侧边栏标题
    setSidebarTitle,     // 设置侧边栏标题
    collapsed,           // 侧边栏是否收起
    toggleCollapsed,     // 切换收起状态
    setCollapsed,        // 直接设置收起状态
    resetSidebar         // 重置侧边栏
  } = useLayout();

  return (
    <div>
      <button onClick={toggleCollapsed}>
        {collapsed ? '展开' : '收起'}侧边栏
      </button>
    </div>
  );
}
```

## 8. 侧边栏集成

### 8.1 基本用法

插件通过 `LayoutContext` 动态注入侧边栏内容：

```typescript
import React, { useEffect } from 'react';
import { useLayout } from '@/shared/context/LayoutContext';

function YourPluginApp() {
  const { setSidebarContent, collapsed } = useLayout();
  const [activeView, setActiveView] = React.useState('home');

  useEffect(() => {
    setSidebarContent(
      <div className="space-y-1 py-2">
        <button
          onClick={() => setActiveView('home')}
          className={`
            w-full flex items-center px-4 py-3 text-[14px] font-medium rounded-lg 
            transition-all duration-200
            ${activeView === 'home' ? 'bg-blue-50 text-blue-600' : 'text-gray-600 hover:bg-gray-50'}
            ${collapsed ? 'justify-center px-0' : ''}
          `}
          title={collapsed ? "首页" : ""}
        >
          <span className={`${collapsed ? 'text-lg w-auto mr-0' : 'w-6 text-center mr-2'} flex items-center justify-center`}>
            <i className="fa-solid fa-house"></i>
          </span>
          {!collapsed && "首页"}
        </button>

        <button
          onClick={() => setActiveView('settings')}
          className={`
            w-full flex items-center px-4 py-3 text-[14px] font-medium rounded-lg 
            transition-all duration-200
            ${activeView === 'settings' ? 'bg-blue-50 text-blue-600' : 'text-gray-600 hover:bg-gray-50'}
            ${collapsed ? 'justify-center px-0' : ''}
          `}
          title={collapsed ? "设置" : ""}
        >
          <span className={`${collapsed ? 'text-lg w-auto mr-0' : 'w-6 text-center mr-2'} flex items-center justify-center`}>
            <i className="fa-solid fa-gear"></i>
          </span>
          {!collapsed && "设置"}
        </button>
      </div>
    );

    // 清理函数：组件卸载时重置侧边栏
    return () => setSidebarContent(null);
  }, [activeView, setSidebarContent, collapsed]);

  return (
    <div className="p-6">
      {activeView === 'home' && <HomeView />}
      {activeView === 'settings' && <SettingsView />}
    </div>
  );
}
```

## 10. 数据持久化

### 10.1 使用 SQLite

推荐使用 SQLite 作为插件的数据库：

```javascript
const sqlite3 = require('sqlite3').verbose();
const path = require('path');

// 数据库文件位于插件根目录
const DB_PATH = path.join(__dirname, '..', 'your-plugin.db');

class Database {
  constructor() {
    this.db = new sqlite3.Database(DB_PATH, (err) => {
      if (err) {
        console.error('[DB] Error opening database:', err);
      } else {
        console.log('[DB] Database connected');
        this.init();
      }
    });
  }

  init() {
    this.db.run(`
      CREATE TABLE IF NOT EXISTS items (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        type TEXT NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
      )
    `);
  }

  // 查询所有
  getAllItems() {
    return new Promise((resolve, reject) => {
      this.db.all('SELECT * FROM items', [], (err, rows) => {
        if (err) reject(err);
        else resolve(rows);
      });
    });
  }

  // 创建
  createItem(name, type) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'INSERT INTO items (name, type) VALUES (?, ?)',
        [name, type],
        function (err) {
          if (err) reject(err);
          else resolve({ id: this.lastID, name, type });
        }
      );
    });
  }

  // 更新
  updateItem(id, data) {
    return new Promise((resolve, reject) => {
      this.db.run(
        'UPDATE items SET name = ?, type = ? WHERE id = ?',
        [data.name, data.type, id],
        (err) => {
          if (err) reject(err);
          else resolve({ id, ...data });
        }
      );
    });
  }

  // 删除
  deleteItem(id) {
    return new Promise((resolve, reject) => {
      this.db.run('DELETE FROM items WHERE id = ?', [id], (err) => {
        if (err) reject(err);
        else resolve({ id });
      });
    });
  }
}

module.exports = new Database();
```

### 10.2 使用 JSON 文件（不推荐用于生产）

对于简单的配置存储，可以使用 JSON 文件：

```javascript
const fs = require('fs');
const path = require('path');

const CONFIG_PATH = path.join(__dirname, '..', 'config.json');

// 读取配置
function readConfig() {
  try {
    const data = fs.readFileSync(CONFIG_PATH, 'utf8');
    return JSON.parse(data);
  } catch (error) {
    return {}; // 默认配置
  }
}

// 写入配置
function writeConfig(config) {
  fs.writeFileSync(CONFIG_PATH, JSON.stringify(config, null, 2));
}
```

 * 计算订阅到期天数
 * @param expiryDate 到期日期（ISO 8601 格式）
 * @returns 剩余天数（负数表示已过期）
 */
function calculateDaysUntilExpiry(expiryDate: string): number {
  const expiry = new Date(expiryDate);
  const now = new Date();
  const diffMs = expiry.getTime() - now.getTime();
  return Math.ceil(diffMs / (1000 * 60 * 60 * 24));
}
